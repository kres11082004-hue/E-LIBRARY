# Athenaeum — E-Library System

A full-stack e-library and physical library management system for educational institutions. Supports 3 user roles (Admin, Instructor, Student), browse/read/save books, personal reading lists, and admin monitoring by campus/course/year/section.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080, proxied at `/api`)
- `pnpm --filter @workspace/elibrary run dev` — run the frontend (port 19545, proxied at `/`)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string, `SESSION_SECRET` — token secret

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite, Wouter router, TanStack Query, shadcn/ui, Tailwind CSS
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Charts: Recharts (monitoring page)

## Where things live

- `lib/db/src/schema/` — DB schema (users, books, my_list, borrow_records, activity_log)
- `lib/api-client-react/src/generated/` — generated hooks + Zod schemas (don't edit manually)
- `artifacts/api-server/src/routes/` — Express routes (auth, users, books, mylist, borrow, monitoring)
- `artifacts/api-server/src/middlewares/auth.ts` — Bearer token auth middleware
- `artifacts/elibrary/src/pages/` — Frontend pages
- `artifacts/elibrary/src/pages/admin/` — Admin-only pages (books, users, monitoring)
- `artifacts/elibrary/src/lib/auth-context.tsx` — Auth context provider
- `artifacts/elibrary/src/components/layout.tsx` — Sidebar navigation layout

## Architecture decisions

- Token-based auth using in-memory `Map<token, userId>` — simple, no JWT library needed
- Auth token stored in localStorage, injected via `setAuthTokenGetter(() => localStorage.getItem("token"))`
- Admin/librarian users auto-approved on register; students/instructors get `isApproved: false`
- Password hashed with SHA-256 + static salt (sufficient for a library system)
- OpenAPI-first: all types and fetch hooks generated by Orval from `artifacts/api-spec/openapi.yaml`

## Product

- **Login/Register** — 3-step wizard for students (role, contact, academic info), 2-step for others
- **Browse Books** — search, filter by category, see physical availability
- **Book Detail** — cover, description, save to list, read online link, physical copy count
- **My Reading List** — personal saved books with offline metadata
- **Dashboard** — greeting, stats (admin), recent books, my list preview
- **Admin: Manage Books** — full CRUD with cover/file URLs, physical copies tracking
- **Admin: Manage Users** — approve/suspend/delete, filter by campus and role
- **Admin: Monitoring** — stats cards, charts by campus, by course/year/section, borrow records, activity feed

## Gotchas

- After editing `artifacts/api-spec/openapi.yaml`, always run codegen: `pnpm --filter @workspace/api-spec run codegen`
- The generated hooks require `queryKey` in `UseQueryOptions`; pass `{ enabled } as any` when only passing `enabled`
- Do NOT run `pnpm dev` at workspace root — start individual workflows instead
- After `pnpm --filter @workspace/db run push`, restart the API server workflow for schema changes to take effect
